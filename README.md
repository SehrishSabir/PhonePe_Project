# 📊 PhonePe Business Analytics Dashboard

## 📌 Project Overview

The **PhonePe Business Analytics Dashboard** is an end-to-end data analytics project developed to analyze customer transactions across multiple financial services, including **Recharge, Money Transfer, Loan, and Insurance**. The project focuses on identifying customer behavior, transaction trends, revenue contribution, and service performance through SQL analysis, Python visualizations, and interactive Power BI dashboards.

---

## 🎯 Business Objective

* Analyze customer transaction patterns across different services.
* Identify high-value customers and top-performing products.
* Monitor payment success and failure trends.
* Understand customer adoption of various services.
* Generate actionable insights to support data-driven business decisions.

---

## 🛠️ Tools & Technologies Used

* **SQL (MySQL)** – Data querying and business analysis
* **Power BI** – Interactive dashboard creation and KPI reporting
* **Python (Pandas, Matplotlib)** – Data cleaning and exploratory data analysis
* **Excel** – Initial data preparation and validation

## 📂 Dataset Description

### 1. All_Users

| Column Name | Description                |
| ----------- | -------------------------- |
| User_ID     | Unique customer identifier |
| Name        | Customer name              |
| Age         | Customer age               |
| Join_Date   | User registration date     |

### 2. Recharge_Bills

| Column Name    | Description           |
| -------------- | --------------------- |
| Transaction_ID | Unique transaction ID |
| User_ID        | Customer identifier   |
| Recharge_Type  | Type of recharge      |
| Amount         | Recharge amount       |
| Date           | Transaction date      |
| Payment_Status | Success/Failed status |
| Reason         | Purpose of recharge   |

### 4. Loan

# 📈 Key Performance Indicators (KPIs)

* Total Users
* Total Transactions
* Total Recharge Amount
* Total Money Transfer Amount
* Total Loan Amount
* Total Insurance Premium
* Payment Success Rate
* Top Customers by Transaction Value

---

# 🔍 Business Questions Solved

1. Identify the top 10 customers contributing the highest transaction value.
2. Determine which age group generates the highest revenue.
3. Find customers using multiple services.
4. Identify the most preferred recharge type.
5. Determine the highest revenue-generating insurance product.
6. Analyze monthly transaction trends.
7. Evaluate payment success and failure rates.
8. Identify top loan borrowers and premium-paying customers.

---

# 📊 Dashboard Features

* Interactive filters and slicers
* Service-wise transaction analysis
* Customer segmentation
* Monthly trend analysis
* Revenue contribution analysis
* Payment performance monitoring

---

# 💡 Key Insights

* Multi-service users contribute significantly more transaction value than single-service users.
* A small group of high-value customers contributes a major portion of total transactions.
* Insurance and Loan services generate higher-value transactions.
* Payment failure analysis helps identify operational improvement opportunities.

---

# 🚀 Business Recommendations

* Target high-value customers through loyalty programs.
* Increase cross-selling of Insurance to Loan customers.
* Improve payment success rates by addressing common failure reasons.
* Launch campaigns for low-adoption services.
